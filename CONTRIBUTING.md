# Contributing

This repository uses pull requests for all changes to `main`.

For this early phase:

1. Create a feature branch.
2. Open a pull request against `main`.
3. Wait for required checks to pass.
4. Keep the branch up to date with `main` when GitHub requires it.
5. Merge only after the required repository rules are satisfied.

Do not commit raw secrets, decrypted Kubernetes secrets, kubeconfigs, tokens,
personal access tokens, or local cluster credentials.
